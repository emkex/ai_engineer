Как устроена страница — минимум для парсинга
HTML · CSS · DOM
HTML — скелет

HTML = дерево тегов. Каждый тег — узел. Теги вкладываются друг в друга. Браузер строит из этого DOM (Document Object Model).

# Ключевые теги для парсинга:
<div class="card">     # контейнер, группировка
<p>текст</p>         # параграф
<a href="url">...</a> # ссылка — .get("href")
<img src="url">       # картинка — .get("src")
<span class="price"> # инлайн-элемент
<table><tr><td>    # таблицы с данными
<ul><li>...        # списки
<h1>...<h6>        # заголовки
Атрибуты тега — это ключи. class, id, href, src, data-* — самые важные для поиска.
CSS селекторы — язык поиска

CSS селекторы — это то, как ты говоришь "найди мне вот этот элемент". BeautifulSoup и Playwright оба понимают CSS селекторы.

Как найти селектор нужного элемента в браузере

Способ 1 — DevTools (F12)

Правая кнопка на элемент → "Просмотреть код" → в панели Elements навести на элемент → правая кнопка → Copy → Copy selector

Скопированный селектор часто избыточный (длинная цепочка). Упрости до класса или id.

Способ 2 — Network tab

F12 → Network → XHR/Fetch → перезагрузить страницу → искать запросы с JSON данными. Часто данные приходят напрямую через API, без парсинга HTML вообще.

Это лучший способ. Нашёл JSON-эндпоинт → парсить HTML не нужно.

Способ 3 — Console JS

F12 → Console → проверить селектор прямо в браузере:

// проверить что находит
document.querySelectorAll('.price')
// текст первого элемента
document.querySelector('.title').innerText

HTTP запросы и библиотека requests
requests · headers · status codes · JSON
Анатомия HTTP запроса и ответа

Запрос состоит из:

ЧастьЧто это
МетодGET (получить) / POST (отправить) / PUT / DELETE
URLадрес + путь + query params после ?
Headersметаданные: кто ты, что принимаешь, токен авторизации
Bodyтолько у POST/PUT — данные которые отправляешь (JSON)

Ответ состоит из:

ЧастьЧто это
Status code200 OK, 404 не найдено, 403 нет доступа, 429 лимит
HeadersContent-Type (json/html), Set-Cookie, X-RateLimit
Bodyсами данные — HTML строка или JSON

URL разбор:

https://api.example.com/v1/news?topic=economics&limit=10

https://          # протокол
api.example.com   # хост
/v1/news          # путь (endpoint)
?topic=economics  # query параметр
&limit=10         # ещё один параметр

Важные статус-коды:

200OK — всё хорошо
201Created — создано (POST)
400Bad Request — ошибка в запросе
401Unauthorized — нужна авторизация
403Forbidden — нет доступа
404Not Found
429Rate Limited — слишком много запросов
500Server Error — ошибка сервера
requests — основные команды
import requests

# GET запрос
r = requests.get('https://httpbin.org/get')
r.status_code   # 200
r.text          # строка (HTML или JSON как строка)
r.content       # байты (для картинок, PDF)
r.json()        # автоматически парсит JSON → dict
r.headers       # заголовки ответа

# GET с параметрами в URL
params = {'topic': 'economics', 'limit': 10}
r = requests.get(url, params=params)
# → url?topic=economics&limit=10

# POST с JSON телом
data = {'query': 'нефть', 'lang': 'ru'}
r = requests.post(url, json=data)
# json= автоматически сериализует и ставит Content-Type

# Всегда проверяй статус
r.raise_for_status()  # бросает исключение если 4xx/5xx
Заголовки — зачем и когда нужны
import requests
from fake_useragent import FakeUserAgent

# Минимальные заголовки для обхода блокировок
headers = {
    'User-Agent': FakeUserAgent().random,
    'Accept': 'application/json',
    'Accept-Language': 'ru-RU,ru;q=0.9',
}

# Авторизация через Bearer токен (Claude API, etc.)
headers = {
    'Authorization': f'Bearer {api_key}',
    'Content-Type': 'application/json',
}

# Session — переиспользует соединение (быстрее)
session = requests.Session()
session.headers.update(headers)
r = session.get(url1)
r = session.get(url2)  # те же заголовки
User-Agent — строка "кто делает запрос". Без неё многие сайты отвечают 403 или пустой страницей.
Паттерн для надёжного парсинга с requests
import requests
import time
from fake_useragent import FakeUserAgent

def safe_get(url: str, retries: int = 3) -> requests.Response | None:
    headers = {'User-Agent': FakeUserAgent().random}
    for attempt in range(retries):
        try:
            r = requests.get(url, headers=headers, timeout=10)
            r.raise_for_status()
            return r
        except requests.HTTPError as e:
            if r.status_code == 429:      # rate limit
                time.sleep(2 ** attempt)  # экспоненциальный backoff
            else:
                break
        except requests.RequestException as e:
            time.sleep(1)
    return None

# Использование
r = safe_get('https://example.com')
if r:
    print(r.json())

Практические паттерны
# Паттерн 1: список карточек → данные
cards = soup.select('.news-card')
results = []
for card in cards:
    title_el = card.select_one('h2')
    link_el  = card.select_one('a')
    if not title_el:
        continue
    results.append({
        'title': title_el.get_text(strip=True),
        'url':   link_el.get('href') if link_el else None,
    })

# Паттерн 2: таблица → список словарей
table = soup.select_one('table')
headers = [th.get_text(strip=True)
           for th in table.select('th')]
rows = []
for tr in table.select('tbody tr'):
    cells = [td.get_text(strip=True)
             for td in tr.select('td')]
    rows.append(dict(zip(headers, cells)))

# Паттерн 3: все ссылки с домена
base = 'https://example.com'
links = {
    a['href'] for a in soup.select('a[href^="/"]')
}
absolute = {base + l for l in links}

RSS парсинг — основной источник данных для "Задумки"

RSS — это XML файл с новостями. Структурированный, без JS, без капч. Идеально для агентов.

import requests
from bs4 import BeautifulSoup as BS

def parse_rss(url: str) -> list[dict]:
    r = requests.get(url, timeout=10)
    # xml парсер для RSS/Atom
    soup = BS(r.content, 'xml')
    items = []
    for item in soup.find_all('item'):
        items.append({
            'title':   item.find('title').text,
            'link':    item.find('link').text,
            'pubDate': item.find('pubDate').text,
            'desc':    item.find('description').text,
        })
    return items

# Примеры RSS для экономики
feeds = [
    'https://feeds.reuters.com/reuters/businessNews',
    'https://www.rbc.ru/rss/news',
]

Playwright — динамические сайты
JS-рендеринг · SPA · автоматизация · скриншоты
Когда нужен Playwright, а когда нет

Playwright нужен:

Страница грузится, но данных нет в HTML → JS подгружает их потом. Нужно кликнуть кнопку "Показать ещё". Нужна авторизация через форму. Нужен скриншот страницы.

Сначала проверь Network tab:

Открой F12 → Network → XHR/Fetch → найди запрос с данными. Если данные приходят через API с JSON — используй requests напрямую. Это быстрее и надёжнее.

Playwright не нужен:

Если данные есть в исходном HTML (Ctrl+U), если есть RSS или API, если нужен просто парсинг статичного контента.

Async Playwright (для агентов)
from playwright.async_api import async_playwright
import asyncio

async def scrape_page(url: str) -> str:
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        page = await browser.new_page()

        # Блокировать картинки — быстрее
        await page.route(
            '**/*.{png,jpg,gif,svg}',
            lambda route: route.abort()
        )

        await page.goto(url, wait_until='domcontentloaded')

        # Ждать конкретный элемент
        await page.wait_for_selector('.article-content')

        html = await page.content()
        await browser.close()
        return html

# Запуск
html = asyncio.run(scrape_page('https://example.com'))

Playwright + BeautifulSoup вместе
# Стандартный паттерн: Playwright рендерит → BS парсит
from playwright.sync_api import sync_playwright
from bs4 import BeautifulSoup as BS

def get_dynamic_data(url: str) -> list[dict]:
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=True)
        page = browser.new_page()
        page.goto(url)
        page.wait_for_load_state('networkidle')
        html = page.content()  # HTML после JS
        browser.close()

    # Теперь парсим как обычно
    soup = BS(html, 'lxml')
    cards = soup.select('.data-card')
    return [{'title': c.get_text(strip=True)} for c in cards]

Как агент использует парсинг

Агент не парсит HTML напрямую. Ты пишешь инструмент (функцию), агент его вызывает когда нужно.

# Инструмент для LangGraph агента
from langchain_core.tools import tool

@tool
def fetch_news(topic: str) -> str:
    """Получает последние новости по теме.
    Используй для получения актуальной информации."""
    items = parse_rss(RSS_FEEDS.get(topic, DEFAULT_FEED))
    return '\n'.join(
        f"{i['title']} — {i['link']}"
        for i in items[:5]
    )

# Агент сам решает когда вызвать
# "найди новости про инфляцию"
# → вызывает fetch_news("инфляция")
# → получает текст → отвечает пользователю
Вся ценность: ты один раз пишешь надёжный парсер, оборачиваешь в инструмент — агент использует его как нужно.

Разбор реального запроса — cookies, headers, params
cURL · перехват · безопасность
Как получить готовый Python код из браузера (cURL)
F12 → Network → XHR/Fetch tab → Нужный запрос → ПКМ → Copy as cURL → curlconverter.com → Готовый requests код

Это называется перехват запроса. Смотришь что браузер отправляет → повторяешь из Python. Так делают неофициальные API-клиенты. XHR/Fetch — типы фоновых запросов: XHR старый способ, Fetch современный. Именно они несут данные (JSON), не картинки и шрифты.

VISITOR_INFOанонимный id — нейтрально
YSCсессия YouTube — протухает
Куки протухают через часы/дни. Хардкодить нельзя. Хранить в .env, добавить в .gitignore.

ЗаголовокСуть

Reverse-engineered API

Официального API нет? Делай свой. Network tab → XHR → найди JSON ответ → Copy as cURL → Python код.

Именно так работают yt-dlp, pytube — нашли внутренние эндпоинты YouTube и повторяют запросы из кода.

RSS — как найти фид и отличить от не-RSS
XML · Atom · детекция
3 способа найти RSS

1 — стандартные пути (открыть в браузере):

site.com/feed
site.com/rss
site.com/feed.xml
site.com/atom.xml

2 — в <head> страницы (Ctrl+U):

<!-- ЭТО RSS — берёшь href -->
<link rel="alternate"
      type="application/rss+xml"
      href="/feed/rss">

<!-- НЕ ЭТО — это поиск сайта -->

Ключевое: rel="alternate" + rss+xml или atom+xml. Значение href — это URL фида.

youtube.com/feeds/videos.xml?channel_id=UCxxx
youtube.com/feeds/videos.xml?playlist_id=PLxxx
RSS vs API vs HTML парсинг

RSS — стандартный XML фид для новостных читалок. Не API. Только читать, не управлять.

API — разработчики сами создают эндпоинты. Документируют. Часто с авторизацией, фильтрами, пагинацией. Двусторонний.

HTML парсинг — последний вариант когда нет ни RSS ни API.

RSS дают не все сайты. Многие русские новостные — нет или дают урезанный. Поэтому иногда парсят HTML.
