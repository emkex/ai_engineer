01 HTTP изнутри протокол · TCP · запрос-ответ

HTTP (HyperText Transfer Protocol) — текстовый протокол поверх TCP. Каждый запрос и ответ это просто текстовые сообщения с чёткой структурой.

Анатомия HTTP запроса

Accept: application/json User-Agent: MyApp/1.0 Python/3.11 Content-Type: application/json ──── (пустая строка = конец заголовков) ────────────── # У GET нет тела. У POST/PUT есть: {"query": "inflation", "lang": "ru"}

Анатомия HTTP ответа

URL разбор — каждая часть имеет смысл

https://api.example.com:443/v1/news?topic=inflation&limit=10#section ├── https:// → протокол (http или https) ├── api.example.com → хост (доменное имя) ├── :443 → порт (443 для HTTPS по умолчанию, 80 для HTTP) ├── /v1/news → путь (endpoint) ├── ? → начало query string ├── topic=inflation → query параметр (key=value) ├── &limit=10 → ещё один параметр (& = разделитель) └── #section → fragment (только в браузере, не отправляется серверу)
⚠ Важно про HTTPS и HTTP

HTTPS = HTTP + TLS шифрование. В HTTPS всё тело запроса зашифровано — провайдер видит только хост, не URL и не тело. Всегда используй HTTPS для API ключей. HTTP = всё открытым текстом.

Идемпотентность — важная концепция

Идемпотентный метод: выполни его 5 раз — результат тот же что от одного выполнения. GET /users/1 можно звать бесконечно — пользователь не размножится. POST /users создаст нового пользователя каждый раз. Это важно при retry логике: GET/PUT/DELETE можно повторять при ошибке, POST — нет.

import requests

# GET — получить данные (нет тела)
r = requests.get('https://jsonplaceholder.typicode.com/posts',
                params={'userId': 1})

# POST — создать (тело — JSON)
r = requests.post('https://jsonplaceholder.typicode.com/posts',
                 json={'title': 'test', 'body': 'text', 'userId': 1})

# PUT — полная замена
r = requests.put('https://jsonplaceholder.typicode.com/posts/1',
                json={'id': 1, 'title': 'new', 'body': 'new body', 'userId': 1})

# PATCH — частичное обновление
r = requests.patch('https://jsonplaceholder.typicode.com/posts/1',
                  json={'title': 'only title changed'})

# DELETE
r = requests.delete('https://jsonplaceholder.typicode.com/posts/1')

401 vs 403 — частая путаница

401: ты не представился. Нет токена или токен неверный. Решение: добавить/поправить токен.
403: ты представился, но тебе сюда нельзя. Токен верный, но прав нет. Решение: другой токен с другими правами.

429 — что делать

Сервер говорит "притормози". Смотри заголовок Retry-After — там секунды до следующей попытки. Или X-RateLimit-Reset — UNIX timestamp. Реализуй exponential backoff.

# Правильная обработка статус-кодов
def handle_response(r: requests.Response) -> dict | None:
    if r.status_code == 200:
        return r.json()
    elif r.status_code == 401:
        raise AuthError("Проверь API ключ")
    elif r.status_code == 429:
        retry_after = int(r.headers.get('Retry-After', 60))
        raise RateLimitError(f"Rate limit. Жди {retry_after}с")
    elif r.status_code >= 500:
        raise ServerError(f"Сервер упал: {r.status_code}")
    else:
        r.raise_for_status()  # всё остальное 4xx — исключение

05 REST принципы архитектура · ресурсы · статelessness

REST (Representational State Transfer) — архитектурный стиль, не протокол. Набор соглашений о том как строить API. Большинство публичных API следуют REST.

6 принципов REST

  • Client-Server — клиент и сервер независимы, общаются через интерфейс
  • Stateless — каждый запрос самодостаточен, сервер не хранит состояние клиента
  • Cacheable — ответы можно кэшировать если сервер разрешил
  • Uniform Interface — единообразные URL и методы
  • Layered System — клиент не знает через сколько слоёв идёт запрос
  • Code on Demand — сервер может отдавать исполняемый код (опционально)

Главное для практики: Stateless

Каждый запрос содержит ВСЁ что нужно серверу для ответа. Нет "памяти" между запросами. Если нужна авторизация — токен передаётся в каждом запросе, не один раз при "входе".

Следствие: API ключ = в каждом запросе. Session в requests не хранит "авторизацию" на сервере — он хранит заголовки локально и отправляет их с каждым запросом.

REST URL соглашения — как читать и строить endpoints

HMAC подпись (ByBit, OKX) — как это работает

Почему биржи не используют просто API Key

Простой Bearer токен: если кто-то перехватил заголовок — всё, у него полный доступ навсегда. HMAC: каждый запрос подписывается уникально используя timestamp + тело запроса. Перехваченная подпись работает только для того конкретного запроса и только в течение нескольких секунд.

import hmac, hashlib, time

def make_bybit_headers(api_key: str, api_secret: str,
                       params: dict) -> dict:
    timestamp = str(int(time.time() * 1000))  # ms
    recv_window = "5000"

    # Строка для подписи: timestamp + key + window + params
    param_str = "&".join(f"{k}={v}" for k, v in sorted(params.items()))
    sign_str  = timestamp + api_key + recv_window + param_str

    # HMAC-SHA256 подпись
    signature = hmac.new(
        api_secret.encode(),
        sign_str.encode(),
        hashlib.sha256
    ).hexdigest()

    return {
        'X-BAPI-API-KEY':     api_key,
        'X-BAPI-TIMESTAMP':   timestamp,
        'X-BAPI-RECV-WINDOW': recv_window,
        'X-BAPI-SIGN':        signature,
    }

Паттерн 1: Класс-клиент для API

Оборачиваешь всё взаимодействие с API в класс. Это то, что делается в уроке про Etherscan, ByBit, ADS Power. Один паттерн — разные API.

from dotenv import load_dotenv
import os, requests, time

load_dotenv()

# Кастомные исключения — из урока про Etherscan
class APIError(Exception): pass
class AuthError(APIError): pass
class RateLimitError(APIError): pass

class BaseAPIClient:
    """Базовый класс для любого REST API клиента."""

    BASE_URL = ""  # перегружать в наследниках

    def __init__(self, api_key: str):
        self._session = requests.Session()
        self._session.headers.update(self._default_headers(api_key))

    def _default_headers(self, api_key: str) -> dict:
        return {
            'Authorization': f'Bearer {api_key}',
            'Accept': 'application/json',
        }

    def _request(self, method: str, endpoint: str,
                  retries: int = 3, **kwargs) -> dict:
        url = self.BASE_URL + endpoint
        for attempt in range(retries):
            try:
                r = self._session.request(method, url,
                                            timeout=10, **kwargs)
                if r.status_code == 401:
                    raise AuthError("Неверный API ключ")
                if r.status_code == 429:
                    wait = int(r.headers.get('Retry-After', 2 ** attempt))
                    time.sleep(wait)
                    continue
                r.raise_for_status()
                return r.json()
            except (AuthError, RateLimitError):
                raise
            except requests.RequestException as e:
                if attempt == retries - 1:
                    raise APIError(f"Запрос упал: {e}") from e
                time.sleep(2 ** attempt)
        raise APIError("Исчерпаны попытки")

    def get(self, endpoint: str, **kwargs) -> dict:
        return self._request('GET', endpoint, **kwargs)

    def post(self, endpoint: str, **kwargs) -> dict:
        return self._request('POST', endpoint, **kwargs)


# Конкретный клиент — Etherscan-подобный
class EtherscanClient(BaseAPIClient):
    BASE_URL = "https://api.etherscan.io/v2/api"

    def _default_headers(self, api_key: str) -> dict:
        # Etherscan ключ идёт в params, не headers
        self._api_key = api_key
        return {'Accept': 'application/json'}

    def get_balance(self, address: str) -> float:
        data = self.get("", params={
            'chainid': 1,
            'module':  'account',
            'action':  'balance',
            'address': address,
            'apikey':  self._api_key,
        })
        return int(data['result']) / 1e18  # wei → ETH

Паттерн 2: Пагинация

Большинство API не отдают сразу тысячи записей. Они разбивают на страницы (pagination). Три основных вида:

# ──────────────────────────────────────────────────────────
# Вид 1: offset/limit (JSONPlaceholder, большинство REST)
# ──────────────────────────────────────────────────────────
def get_all_offset(base_url: str, limit: int = 100) -> list:
    results, offset = [], 0
    while True:
        r = requests.get(base_url, params={'_start': offset, '_limit': limit})
        page = r.json()
        if not page:  # пустая страница = конец
            break
        results.extend(page)
        offset += limit
    return results

# ──────────────────────────────────────────────────────────
# Вид 2: page/per_page (GitHub API, многие REST)
# ──────────────────────────────────────────────────────────
def get_all_pages(url: str) -> list:
    results, page = [], 1
    while True:
        r = requests.get(url, params={'page': page, 'per_page': 100})
        data = r.json()
        if not data:
            break
        results.extend(data)
        # Многие API отдают total в заголовке
        total = r.headers.get('X-Total-Count')
        if total and len(results) >= int(total):
            break
        page += 1
    return results

# ──────────────────────────────────────────────────────────
# Вид 3: cursor (Twitter/X API, Etherscan txlist)
# ──────────────────────────────────────────────────────────
def get_all_cursor(url: str, params: dict) -> list:
    results = []
    while True:
        r = requests.get(url, params=params)
        data = r.json()
        results.extend(data['items'])
        next_cursor = data.get('next_cursor')
        if not next_cursor:
            break
        params['cursor'] = next_cursor
    return results
09 Ошибки и Retry логика exceptions · exponential backoff · jitter 
import requests, time, random

# Иерархия исключений requests
# RequestException
#   ├── ConnectionError     — сеть недоступна, DNS упал
#   ├── Timeout             — connect или read timeout
#   │   ├── ConnectTimeout
#   │   └── ReadTimeout
#   └── HTTPError           — raise_for_status() при 4xx/5xx

def safe_request(method: str, url: str,
                  max_retries: int = 3,
                  backoff_base: float = 1.0,
                  **kwargs) -> requests.Response:
    """
    Надёжный HTTP запрос с exponential backoff + jitter.
    Повторяет только при: ConnectionError, Timeout, 429, 5xx.
    НЕ повторяет: 4xx (кроме 429) — это ошибка запроса, не сети.
    """

    RETRIABLE_STATUS = {429, 500, 502, 503, 504}

    for attempt in range(max_retries + 1):
        try:
            r = requests.request(method, url, timeout=10, **kwargs)

            if r.status_code in RETRIABLE_STATUS and attempt < max_retries:
                # Для 429 читаем Retry-After
                wait = float(r.headers.get('Retry-After',
                               backoff_base * (2 ** attempt)))
                # Jitter: добавляем случайность чтобы не все клиенты
                # стучались одновременно после общего сбоя
                jitter = random.uniform(0, wait * 0.1)
                time.sleep(wait + jitter)
                continue

            r.raise_for_status()
            return r

        except (requests.ConnectionError, requests.Timeout) as e:
            if attempt == max_retries:
                raise
            wait = backoff_base * (2 ** attempt) + random.uniform(0, 0.1)
            time.sleep(wait)

    raise requests.RequestException("Max retries exceeded")
10 Rate Limiting лимиты · throttling · мониторинг

Все серьёзные API имеют лимиты. Anthropic: 50 req/min для бесплатного. Etherscan: 5 req/sec (бесплатный tier). ByBit: 120 req/min. Если не соблюдать — 429 и временный бан.

import time
from collections import deque

class RateLimiter:
    """Sliding window rate limiter."""

    def __init__(self, max_calls: int, period: float):
        self.max_calls = max_calls
        self.period    = period  # секунды
        self._calls    = deque()

    def __call__(self):
        now = time.monotonic()
        # Удалить старые вызовы за пределами окна
        while self._calls and self._calls[0] < now - self.period:
            self._calls.popleft()
        # Если лимит — ждём
        if len(self._calls) >= self.max_calls:
            sleep_time = self.period - (now - self._calls[0])
            if sleep_time > 0:
                time.sleep(sleep_time)
        self._calls.append(time.monotonic())

# Использование
limiter = RateLimiter(max_calls=5, period=1.0)  # 5 req/sec

for address in addresses:
    limiter()  # блокирует если слишком быстро
    balance = client.get_balance(address)

curl-cffi из урока Etherscan — зачем

Некоторые сайты блокируют обычный requests через fingerprinting (отличают Python от браузера по TLS handshake). curl-cffi имитирует fingerprint браузера (Chrome, Firefox). API как Etherscan этого не требуют — там обычный requests достаточен. curl-cffi нужен для сайтов с Cloudflare и продвинутой защитой.

        r = await session.get(url, impersonate="chrome")
        return r.json()
