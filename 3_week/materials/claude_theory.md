**curl из терминала или из Python** — верно. Документация показывает оба варианта: curl напрямую к `https://api.anthropic.com/v1/messages` с заголовками вручную, и Python SDK через `Anthropic()` клиент.

**SDK = готовый способ обращаться, включает автоматизацию заголовков** — верно. SDK автоматически управляет заголовками `x-api-key`, `anthropic-version`, `content-type`, плюс добавляет retry логику, обработку ошибок, стриминг и таймауты. Без SDK всё это пишешь руками через requests.

**Через requests тоже можно, просто передаёшь заголовки сам** — верно. Каждый запрос требует три обязательных заголовка: `x-api-key`, `anthropic-version`, `content-type: application/json`. SDK просто убирает эту рутину.

**REST API = про эндпоинты, Stateless** — верно. Claude API прямо описан как RESTful API. Основной эндпоинт `POST /v1/messages`. Каждый запрос независимый — если не передаёшь историю сообщений в теле запроса, модель не знает что было раньше.

**Свои классы поверх SDK всё равно нужны** — верно. SDK даёт универсальные примитивы, а твой `JSONPlaceholderClient` из файла нед.3 — это именно правильный паттерн: обернул клиент в dataclass, инкапсулировал session, написал методы под свои эндпоинты. Для Anthropic будет то же самое: обернёшь `Anthropic()` в свой класс с методами `evaluate_news()`, `summarize()` и т.д.

Понимание полное, можно писать код.
