# ООП

# Классы и объекты

**Объектно-ориентированное программирование (ООП)** — это подход, при котором программа рассматривается как набор объектов, взаимодействующих друг с другом. У каждого есть свойства и поведение. ООП ускоряет написание кода и делает его более читаемым.

**Классы** - это как чертежи или шаблоны, которые определяют, как создавать объекты. Вы можете думать о классе как о наборе инструкций о том, как создать что-то определенное. Также классы можно рассматривать как пользовательские типы данных. 

**Объекты** - это экземпляры классов. Они создаются на основе этих классов. Когда вы создаете объект, вы используете класс как "форму" для создания экземпляра этого объекта.

# Создание класса и объекта

## Создание класса

Самый простой класс в Python можно создать с использованием ключевого слова `class`, после которого идет имя класса, и двоеточие. Внутри класса вы можете определить атрибуты (переменные) и методы (функции), которые характеризуют этот класс.

```python
class MyClass:
    pass
```

Этот класс называется `MyClass`, и он не имеет никаких атрибутов или методов (пустой). Ключевое слово `pass` используется в Python, когда синтаксически требуется наличие инструкции, но никакого действия выполнять не нужно.

Вы также можете добавить атрибуты и методы к классу. Например:

```python
import random

class Human:
    def __init__(self, name: str, age: int):
        self.name = name
				self.age = age
				self.height = random.randint(150, 200)

    def show_info(self):
        print(f"name: {self.name}; age: {self.age}; height: {self.height}")
```

Этот класс `Human` имеет атрибуты `name` , `age`, `height` и метод `show_info()`

Метод `__init__()` является конструктором класса, который вызывается при создании нового экземпляра класса

Метод `show_info()` показывает информацию о человеке: имя, возраст и рост (имя и возраст передаются в конструктор, а рост генерируется рандомно)

### `self`

`self` в Python - это ссылка на текущий экземпляр объекта класса. Когда вы создаете метод в классе, первым параметром всегда должен быть `self`, который представляет текущий экземпляр объекта

Простыми словами, `self` говорит Python, что вы обращаетесь к атрибутам и методам текущего объекта. Без `self` Python не сможет определить, к какому объекту применить метод или какой атрибут использовать

`self` используется для доступа к атрибутам и методам объекта внутри самого класса. Когда вы вызываете метод объекта, Python автоматически передает этот объект в метод через параметр `self`, что позволяет методам работать с данными конкретного экземпляра этого класса

## Создание объекта

```python
john = Human(name='John', age=29)
bob = Human(name='Bob', age=40)
alice = Human(name='Alice', age=22)

alex.show_info()  # name: John; age: 29; height: 173
alice.show_info()  # name: Alice; age: 22; height: 191
```

# Статические поля и методы

## Статические поля

Статическое поле - это атрибут, который принадлежит классу в целом, а не экземплярам этого класса. В вашем примере, если бы у вас было статическое поле, например, `population`, оно было бы общим для всех экземпляров класса `Human`. Вы могли бы использовать его для отслеживания общего числа людей в вашей программе.

```python
class Human:
    population = 0  # статическое поле

    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age
        self.height = random.randint(150, 200)
        Human.population += 1  # Увеличиваем общее число людей при создании нового экземпляра класса

    def show_info(self):
        print(f"name: {self.name}; age: {self.age}; height: {self.height}")

# Создаем несколько экземпляров класса Human
person1 = Human("Alice", 30)
person2 = Human("Bob", 25)

print("Total population:", Human.population)  # Выводим общее число людей
```

## Статические методы

Статический метод - это метод, который принадлежит классу, а не экземплярам этого класса. Он не имеет доступа к атрибутам экземпляра и может использоваться без создания экземпляра класса.

```python
class Human:
    @staticmethod
    def adult_age(age: int) -> bool:
        return age >= 18

# Использование статического метода без создания экземпляра класса
print(Human.adult_age(20))  # Выведет True
print(Human.adult_age(15))  # Выведет False
```

# Магические методы

Магические методы в Python - это специальные методы, которые начинаются и заканчиваются двойным подчеркиванием (например, `__init__`, `__str__`, `__add__`). Они также известны как "специальные методы" или "методы-магические", потому что они позволяют реализовывать различные операции или функциональность, которая неявно вызывается во время выполнения различных операций в Python.

Магические методы позволяют определить поведение объектов в различных контекстах, таких как инициализация объекта, преобразование объекта в строку, выполнение математических операций с объектами, доступ к элементам объекта и многое другое.

1. `__init__(self, ...)`: Инициализация объекта. Этот метод вызывается при создании нового экземпляра класса.
2. `__str__(self)`: Преобразование объекта в строку. Вызывается при использовании функции `str()` или при использовании объекта в форматированных строках.
3. `__repr__(self)`: Представление объекта. Этот метод вызывается функцией `repr()` для получения представления объекта в виде строки.
4. `__add__(self, other)`: Сложение объектов. Вызывается при использовании оператора `+` для сложения объектов.
5. `__len__(self)`: Получение длины объекта. Вызывается функцией `len()` для получения длины объекта.
6. `__getitem__(self, key)`: Получение элемента по ключу. Вызывается при использовании оператора индексации `[]` для доступа к элементам объекта.
7. `__eq__(self, other)`: Проверка на равенство. Вызывается при использовании оператора сравнения `==`.
8. `__lt__(self, other)`, `__gt__(self, other)`, `__le__(self, other)`, `__ge__(self, other)`: Сравнение объектов. Вызывается при использовании операторов сравнения (`<`, `>`, `<=`, `>=`).
9. `__del__(self)`: Удаление объекта. Вызывается при удалении объекта из памяти.

## Деструктор

Деструктор в объектно-ориентированном программировании - это специальный метод, который вызывается при удалении объекта или когда он выходит из области видимости. В Python деструктор обычно называется методом **`__del__()`**.

Деструктор используется для выполнения очистки или освобождения ресурсов, связанных с объектом, перед его удалением из памяти. Это может включать в себя освобождение памяти, закрытие файлов, отключение соединений с базой данных и т. д.

```python
class Human:
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age
        self.height = random.randint(150, 200)

    def __del__(self):
        print(f"{self.name} is being destroyed")

    def show_info(self):
        print(f"name: {self.name}; age: {self.age}; height: {self.height}")

# Создаем экземпляр класса Human
person = Human("Alice", 30)
person.show_info()

# Удаляем экземпляр класса Human
del person  # Это приведет к вызову деструктора
```

## Методы `__str__()` и `__repr__()`

`__str__()` и `__repr__()` - это два магических метода в Python, которые используются для представления объектов в виде строк.

`__str__()`: Этот метод вызывается функцией `str()` и используется для представления объекта в читаемом формате для человека. Он должен возвращать строку, которая содержит информацию об объекте в удобочитаемом формате. Это часто используется для вывода информации о объекте на экран или в журнале.

```python
class Human:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __str__(self):
        return f"Human: name={self.name}, age={self.age}"

# Создаем объект класса Human
person = Human("Alice", 30)

# Вызываем функцию str() для объекта
print(str(person))  # Выведет: Human: name=Alice, age=30
```

`__repr__()`: Этот метод вызывается функцией `repr()` и используется для представления объекта в виде строки, которая может быть использована для воссоздания этого объекта. Обычно `__repr__()` возвращает строку, которая может быть передана функции `eval()` для создания точной копии объекта. Во многих случаях реализация `__repr__()` совпадает с реализацией `__str__()`, но в отличие от `__str__()`, `__repr__()` часто используется для отладки и регистрации.

```python
class Human:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __repr__(self):
        return f"Human('{self.name}', {self.age})"

# Создаем объект класса Human
person = Human("Alice", 30)

# Вызываем функцию repr() для объекта
print(repr(person))  # Выведет: Human('Alice', 30)
```

## Метод `__call__()`

В Python метод `__call__()` является магическим методом, который позволяет вызывать экземпляр класса как функцию. При вызове экземпляра класса как функции, интерпретатор Python автоматически вызывает метод `__call__()` этого экземпляра.

```python
class Human:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __call__(self, action):
        print(f"{self.name} is {action}")

# Создаем экземпляр класса Human
person = Human("Alice", 30)

# Вызываем экземпляр класса как функцию, что вызовет метод __call__()
person("talking")  # Выведет: Alice is talking
```

## Методы для сравнения

1. `__eq__(self, other)`: Проверка на равенство (==).
2. `__ne__(self, other)`: Проверка на неравенство (!=).
3. `__lt__(self, other)`: Проверка на меньше (<).
4. `__le__(self, other)`: Проверка на меньше или равно (<=).
5. `__gt__(self, other)`: Проверка на больше (>).
6. `__ge__(self, other)`: Проверка на больше или равно (>=).

Методы сравнения - это магические методы, которые используются для определения поведения операторов сравнения. Когда объекты сравниваются этими операторами, интерпретатор Python автоматически вызывает нужный метод для первого объекта, передавая второй объект в качестве аргумента.

```python
class Human:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __eq__(self, other):
        if isinstance(other, Human):
            return self.name == other.name and self.age == other.age
        return False

    def __lt__(self, other):
        if isinstance(other, Human):
            return self.age < other.age
        return NotImplemented

    def __gt__(self, other):
        if isinstance(other, Human):
            return self.age > other.age
        return NotImplemented

    def __le__(self, other):
        if isinstance(other, Human):
            return self.age <= other.age
        return NotImplemented

    def __ge__(self, other):
        if isinstance(other, Human):
            return self.age >= other.age
        return NotImplemented

    def __ne__(self, other):
        return not self.__eq__(other)

# Создаем несколько экземпляров класса Human
person1 = Human("Alice", 30)
person2 = Human("Bob", 25)
person3 = Human("Alice", 30)

# Сравниваем объекты
print(person1 == person2)  # Выведет False
print(person1 == person3)  # Выведет True
print(person1 < person2)   # Выведет False
print(person1 > person2)   # Выведет True
print(person1 <= person2)  # Выведет False
print(person1 >= person2)  # Выведет True
```

## Математические методы

1. `__add__(self, other)`: Сложение (`+`).
2. `__sub__(self, other)`: Вычитание (`-`).
3. `__mul__(self, other)`: Умножение (`*`).
4. `__truediv__(self, other)`: Деление (`/`).
5. `__floordiv__(self, other)`: Целочисленное деление (`//`).
6. `__mod__(self, other)`: Остаток от деления (`%`).
7. `__pow__(self, other)`: Возведение в степень (`*`).
8. `__abs__(self)`: Модуль числа (`abs()`).

Математические методы - это магические методы, которые используются для определения поведения метематических операторов. Когда мы используем математические операторы с объектами данного класса, интерпретатор Python автоматически вызывает нужный метод для первого объекта, передавая второй объект в качестве аргумента.

```python
class Human:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __add__(self, other):
        if isinstance(other, Human):
            return self.age + other.age
        return NotImplemented

    def __sub__(self, other):
        if isinstance(other, Human):
            return self.age - other.age
        return NotImplemented

    def __mul__(self, other):
        if isinstance(other, Human):
            return self.age * other.age
        return NotImplemented

    def __truediv__(self, other):
        if isinstance(other, Human):
            return self.age / other.age
        return NotImplemented

    def __floordiv__(self, other):
        if isinstance(other, Human):
            return self.age // other.age
        return NotImplemented

    def __mod__(self, other):
        if isinstance(other, Human):
            return self.age % other.age
        return NotImplemented

    def __pow__(self, other):
        if isinstance(other, Human):
            return self.age ** other.age
        return NotImplemented

    def __abs__(self):
        return abs(self.age)

# Создаем два объекта класса Human
person1 = Human("Alice", 30)
person2 = Human("Bob", 25)

# Выполняем математические операции
print(person1 + person2)  # Выведет: 55 (сумма возрастов)
print(person1 - person2)  # Выведет: 5 (разница возрастов)
print(person1 * person2)  # Выведет: 750 (произведение возрастов)
print(person1 / person2)  # Выведет: 1.2 (деление возрастов)
print(person1 // person2) # Выведет: 1 (целочисленное деление возрастов)
print(person1 % person2)  # Выведет: 5 (остаток от деления возрастов)
print(person1 ** person2) # Выведет: 847288609443 (возведение возраста person1 в степень возраста person2)
print(abs(person1 - person2))  # Выведет: 5 (модуль разности возрастов)
```

# Основные принципы ООП

Основные принципы ООП включают:

1. **Инкапсуляция**: Объединение данных и методов, которые работают с этими данными, в единый объект. Это позволяет скрыть детали реализации и предоставить интерфейс для взаимодействия с объектом.
2. **Наследование**: Возможность создания новых классов на основе существующих (родительских) классов. Новый класс наследует свойства и методы родительского класса и может добавлять к ним новый функционал или изменять существующий.
3. **Полиморфизм**: Возможность использования объектов разных классов через общий интерфейс. Это означает, что методы могут иметь разные реализации в разных классах, но обращаться к ним можно одинаково.
4. **Абстракция**: Отделение сущности от ее реализации. Абстракция позволяет сосредоточиться на ключевых аспектах объекта и игнорировать детали, которые не важны для данного контекста.

# Основные принципы ООП простыми словами

## **Инкапсуляция**

![Untitled](https://prod-files-secure.s3.us-west-2.amazonaws.com/e267214b-0f78-4543-8f26-ce8e63b9760e/13a469ce-9968-4c01-977e-d2987fe582df/Untitled.png)

У кофемолки есть несколько основных компонентов: механизм для перемалывания кофейных зерен, контейнер для зерен, контейнер для молотого кофе и кнопки или рычаги для управления процессом. Для создания инкапсуляции мы можем объединить эти компоненты в единый объект "кофемолка".

Суть скрытия деталей реализации заключается в том, что пользователь (или другой код) может использовать кофемолку, нажимая на кнопку, чтобы получить молотый кофе, но он не должен иметь доступа к внутренней структуре кофемолки. Например, пользователь не должен иметь возможности как-то взаимодействовать с механизмом перемалывания кофейных зерен.

Вместо этого пользователь может взаимодействовать с кофемолкой только через ее публичный интерфейс - кнопку. Он нажимает на кнопку, кофемолка молет кофейные зерна, и пользователь получает молотый кофе без необходимости знать, как именно это происходит внутри кофемолки.

## **Наследование**

![Untitled](https://prod-files-secure.s3.us-west-2.amazonaws.com/e267214b-0f78-4543-8f26-ce8e63b9760e/c04934a7-92ed-4100-a9aa-3061665083a9/Untitled.png)

Представим себе класс кофемолки и класс кофеварки. Кофемолка уже обладает функциональностью молоть кофейные зерна, а кофеварка должна иметь возможность и молоть кофейные зерна, и варить напиток.

Когда мы говорим о наследовании в этом контексте, мы можем представить себе, что класс кофеварки "наследует" функциональность класса кофемолки, чтобы получить возможность молоть кофейные зерна. Таким образом, кофеварка использует уже существующий функционал, который уже реализован в классе кофемолки.

Затем кофеварка дополняет этот функционал собственным: она добавляет функцию варки кофе. Это означает, что мы можем использовать и функциональность молоть кофейные зерна (унаследованную от кофемолки), и функциональность варить кофе (добавленную в класс кофеварки).

Таким образом, наследование позволяет нам повторно использовать код, избегая дублирования, и добавлять новый функционал, не изменяя уже существующий, что делает код более эффективным и легко поддерживаемым.

## Полиморфизм

![Untitled](https://prod-files-secure.s3.us-west-2.amazonaws.com/e267214b-0f78-4543-8f26-ce8e63b9760e/02e0ec34-ef7b-4f1e-a448-5bf46fd19e48/Untitled.png)

Полиморфизм в объектно-ориентированном программировании означает, что различные классы могут иметь одинаковые методы, но они могут реализовывать эти методы по-разному. Другими словами, один и тот же метод может иметь различное поведение в зависимости от типа объекта, который его вызывает.

Полиморфизм в контексте оружия означает, что разные виды оружия могут иметь общий метод, но реализовывать его по-разному в зависимости от их типа. Давайте рассмотрим пример с пистолетом и базукой:

У нас есть базовый класс `Weapon` (Оружие), у которого есть метод `fire()` (Выстрел). Этот метод будет выполнять действие стрельбы, но конкретная реализация этого действия будет различаться в зависимости от типа оружия.

Класс `Pistol` (Пистолет) переопределяет метод `fire()` таким образом, чтобы он выполнял одиночный выстрел.

Класс `Bazooka` (Базука), с другой стороны, переопределяет метод `fire()` таким образом, чтобы он выполнял мощный взрыв или стрелял ракетой.

Таким образом, мы можем видеть, что оба класса (`Pistol` и `Bazooka`) используют метод `fire()`, но реализация этого метода различна в каждом из них. Это и есть пример полиморфизма: разные объекты (типы оружия) могут использовать один и тот же метод (`fire()`), но его поведение будет разным в каждом случае, соответствуя функциональности конкретного типа оружия.

## Абстракция

![Untitled](https://prod-files-secure.s3.us-west-2.amazonaws.com/e267214b-0f78-4543-8f26-ce8e63b9760e/c3588921-a431-4cc7-8a1c-fb7c72b2b34c/Untitled.png)

Абстракция в программировании означает скрытие деталей реализации и предоставление пользователю только необходимой информации или интерфейса для взаимодействия с объектом. Другими словами, это означает концентрацию на важных аспектах объекта, игнорируя детали, которые не являются существенными для данного контекста.

Когда мы едем на машине, нам не нужно знать, как именно работает двигатель, какие запчасти в нем используются и как именно происходит процесс внутреннего сгорания. Нам нужно знать только, как управлять машиной: как включить двигатель, как пользоваться педалями газа, тормоза и сцепления, как поворачивать руль и т.д.

В этом случае абстракция заключается в том, что мы скрываем детали реализации (внутреннее устройство двигателя, механизмы передачи и т.д.) и предоставляем пользователю простой интерфейс для взаимодействия с машиной (руль, педали, рычаги). Мы абстрагируемся от сложных деталей работы машины и сосредотачиваемся только на том, как ее использовать.

Таким образом, абстракция позволяет упростить сложные системы, делая их более понятными и легкими в использовании, и предоставляет пользователю только необходимую информацию или функционал для достижения конкретных целей.

# Основные принципы ООП в коде

## Инкапсуляция

Инкапсуляция в объектно-ориентированном программировании (ООП) - это механизм, который ограничивает доступ к некоторым компонентам объекта, скрывая детали его реализации от внешнего мира. В Python инкапсуляция реализуется с использованием подчеркиваний.

1. Одно подчеркивание `_`:
    - Поле или метод, начинающийся с одного подчеркивания, считается "приватным" или "защищенным". Однако в Python это всего лишь соглашение между разработчиками, а не строгая проверка доступа.
    - Поля и методы с одним подчеркиванием не рекомендуется использовать за пределами класса, но доступ к ним всё же возможен.
    - Это скорее сигнал для других программистов о том, что поле или метод предназначен для внутреннего использования в классе, а не для внешнего доступа.
2. Два подчеркивания `__`:
    - Поле или метод, начинающийся с двух подчеркиваний, считается "приватным" и имеет иное поведение.
    - Python использует механизм именования "name mangling" для изменения имени таких полей и методов внутри класса, чтобы избежать конфликтов имен с другими классами.
    - Поля и методы с двумя подчеркиваниями становятся недоступными за пределами класса.
    - Это обеспечивает более сильную инкапсуляцию, так как предотвращает нежелательный доступ к полям и методам извне класса.
    
    ```python
    class Human:
        def __init__(self, name, age):
            self.name = name  # публичное поле
            self._age = age   # защищенное поле
            self.__height = 160  # приватное поле
    
        def get_height(self):
            return self.__height
    
        def set_height(self, height):
            if height > 0:
                self.__height = height
    
    # Создаем объект класса Human
    person = Human("Alice", 30)
    
    # Публичное поле доступно извне
    print(person.name)  # Выведет: Alice
    
    # Защищенное поле доступно извне, но соглашение говорит, что оно предназначено для внутреннего использования
    print(person._age)  # Выведет: 30
    
    # Приватное поле недоступно извне
    # print(person.__height)  # Ошибка: AttributeError: 'Human' object has no attribute '__height'
    
    # Однако его можно получить и изменить, используя "name mangling"
    print(person._Human__height)  # Выведет: 160
    person._Human__height = 170
    print(person._Human__height)  # Выведет: 170
    
    # Лучше использовать геттеры и сеттеры для доступа к приватным полям
    print(person.get_height())  # Выведет: 170
    person.set_height(175)
    print(person.get_height())  # Выведет: 175
    
    # Список всех полей и методов объекта можно увидеть с помощью функции dir(obj)
    print(dir(person))  # ['_Human__height', '__class__', '__delattr__', '__dict__', '__dir__', '__doc__', '__eq__', '__format__', '__ge__', '__getattribute__', '__getstate__', '__gt__', '__hash__', '__init__', '__init_subclass__', '__le__', '__lt__', '__module__', '__ne__', '__new__', '__reduce__', '__reduce_ex__', '__repr__', '__setattr__', '__sizeof__', '__str__', '__subclasshook__', '__weakref__', '_age', 'get_height', 'name', 'set_height']
    ```
    

## Наследование

Наследование в объектно-ориентированном программировании (ООП) позволяет создавать новый класс на основе уже существующего класса. Класс, который наследует функциональность другого класса, называется "подклассом" или "наследником", а класс, от которого наследуются свойства и методы, называется "родительским классом" или "суперклассом".

```python
class Human:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        print(f"Hello, my name is {self.name} and I am {self.age} years old.")

# Класс Student наследует функциональность класса Human
class Student(Human):
    def __init__(self, name, age, student_id):
        # Вызываем конструктор родительского класса, чтобы инициализировать общие поля
        super().__init__(name, age)
        self.student_id = student_id

    def study(self, subject):
        print(f"{self.name} is studying {subject}.")

# Создаем объекты классов Human и Student
person = Human("Alice", 30)
student = Student("Bob", 20, "S12345")

# Вызываем методы объектов
person.introduce()  # Выведет: Hello, my name is Alice and I am 30 years old.
student.introduce() # Выведет: Hello, my name is Bob and I am 20 years old.
student.study("Math")  # Выведет: Bob is studying Math.
```

В этом примере класс `Student` наследует функциональность класса `Human`. Класс `Student` расширяет класс `Human`, добавляя новые атрибуты и методы, такие как `student_id` и `study()`. При этом класс `Student` сохраняет все функции и методы класса `Human`, такие как `introduce()`.

### Метод `super()`

`super()` в Python используется для обращения к методам родительского класса в дочернем классе, позволяя вызывать методы и использовать атрибуты родительского класса в переопределенных методах дочернего класса.

Основная задача `super()` - обеспечить возможность доступа к родительским методам и атрибутам в случае, когда методы в дочернем классе имеют такое же имя, как и методы в родительском классе

## Полиморфизм

Полиморфизм в объектно-ориентированном программировании (ООП) означает, что один и тот же метод может вести себя по-разному в разных классах, наследующих один и тот же интерфейс или базовый класс. Это означает, что методы могут быть вызваны с одинаковыми именами, но выполнять различные действия в зависимости от типа объекта, с которым они вызываются.

```python
class Weapon:
    def fire(self):
        pass

class Pistol(Weapon):
    def fire(self):
        print("Pistol fires a bullet")

class Bazooka(Weapon):
    def fire(self):
        print("Bazooka fires a rocket")

class Human:
    def shoot(self, weapon):
        weapon.fire()

# Создаем экземпляры классов
pistol = Pistol()
bazooka = Bazooka()
human = Human()

# Вызываем метод shoot() объекта human, передавая ему объекты разных классов Weapon
human.shoot(pistol)   # Выведет: Pistol fires a bullet
human.shoot(bazooka)  # Выведет: Bazooka fires a rocket
```

В этом примере класс `Weapon` представляет базовый класс, у которого есть метод `fire()`. Классы `Pistol` и `Bazooka` наследуют этот метод и переопределяют его, чтобы реализовать специфическое поведение стрельбы для каждого типа оружия.

Класс `Human` имеет метод `shoot()`, который принимает в качестве параметра объект типа `Weapon` и вызывает его метод `fire()`. При вызове метода `shoot()` объекта `human` с различными объектами `Weapon` (`pistol` и `bazooka`) он вызывает соответствующие методы `fire()` для каждого типа оружия, что иллюстрирует полиморфизм. Важно отметить, что метод `fire()` ведет себя по-разному в разных классах, реализующих его, но он вызывается с одним и тем же именем.
